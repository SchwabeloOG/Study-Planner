How to import from Schoology 📖

Two reliable ways to get your assignments in — pick whichever works.

🔑 Before you start

Schoology doesn't offer a public API for students, but every account has a personal iCal feed — a read-only URL that contains all your calendar events (assignments, tests, due dates). That's what StudyDeck imports.

  • The feed URL is unique to your account — don't share it publicly.
  • It's read-only, so StudyDeck can't accidentally change anything in Schoology.
  • Re-importing later only adds new items — no duplicates.

⭐⭐ Best method: Google Apps Script bridge

A tiny Google Apps Script runs as you and serves your calendar data as JSON. No API key, no CORS issues, no calendar-public requirement. Takes ~3 minutes to set up.

  1. Subscribe to Schoology in Google Calendar first. calendar.google.com → "Other calendars" + → From URL → paste Schoology iCal URL. Wait a minute for Google to pull events.
  2. Find the subscribed calendar's ID. In Google Calendar, click the subscribed calendar → Settings → scroll to Integrate calendar → copy the Calendar ID (looks like abc123@import.calendar.google.com).
  3. Create the Apps Script. Go to script.google.com → New project. Delete any sample code and paste:
    function doGet() { const calId = "b00bjq04ro7idicti1o6oce21odk43j7@import.calendar.google.com"; const calendar = CalendarApp.getCalendarById(calId); if (!calendar) { return ContentService.createTextOutput(JSON.stringify({error: "Calendar not found. Check the ID."})) .setMimeType(ContentService.MimeType.JSON); } // Look from 1 month ago to 1 year in the future to be safe const start = new Date(); start.setMonth(start.getMonth() - 1); const end = new Date(); end.setFullYear(end.getFullYear() + 1); const events = calendar.getEvents(start, end); const output = events.map(e => ({ title: e.getTitle(), start: e.getStartTime(), end: e.getEndTime(), description: e.getDescription(), location: e.getLocation() })); return ContentService.createTextOutput(JSON.stringify(output)) .setMimeType(ContentService.MimeType.JSON); } function testPermissions() { const calId = "b00bjq04ro7idicti1o6oce21odk43j7@import.calendar.google.com"; const calendar = CalendarApp.getCalendarById(calId); if (calendar) { Logger.log("Success! Found calendar: " + calendar.getName()); const events = calendar.getEvents(new Date(), new Date(Date.now() + 86400000)); Logger.log("Events found in next 24h: " + events.length); } else { Logger.log("Failed to find calendar. Check the ID."); } } }
    Replace PASTE_YOUR_CALENDAR_ID_HERE with the ID you copied.
  4. Deploy it. Click Deploy → New deployment. Click the gear icon → select Web app. Fill in:
    • Execute as: Me
    • Who has access: Anyone
    Click Deploy. The first time you'll need to Authorize access (click through the "unsafe" warning — it's your own script). Copy the Web App URL (ends in /exec).
  5. Paste into StudyDeck. Settings → Apps Script bridge → paste the URL → Save → 🔄 Sync via Apps Script. Done.
Why it's the best: The script runs with your Google permissions, so it can read any calendar you've subscribed to — no need to make it public. It returns JSON, which browsers handle without CORS. And to update later, just re-run Sync — the script pulls the latest events from Google (which pulled from Schoology a few hours earlier).
Want more history? Change 90 in the script to 365 to pull a full year, then click Deploy → Manage deployments → pencil icon → New version → Deploy.

⭐ Alternative: Google Calendar API key

This is the most reliable method because it bypasses the CORS block that most schools have. The idea:

  1. Subscribe to Schoology in Google Calendar. Go to calendar.google.com → left sidebar "Other calendars" → + → From URL. Paste your Schoology iCal URL. Google fetches and refreshes it every few hours.
  2. Make that calendar public. Click the three-dot menu next to it → Settings and sharing. Under "Access permissions" check Make available to public (see only free/busy is fine — we need the events). Save.
  3. Copy the Calendar ID. Scroll down to Integrate calendar and copy the long Calendar ID (looks like abc123@group.calendar.google.com).
  4. Create a free Google API key. Go to console.cloud.google.com/apis/credentials, create a project, enable the Google Calendar API, then create an API key. Copy it.
  5. Paste both into StudyDeck. Settings → 🎓 Schoology sync → Google Calendar bridge. Paste key + ID. Click Save then Sync from Google. That's it.
Why it works: Google's servers (not your browser) fetch from Schoology. Google's API supports CORS, so StudyDeck can read it from your browser without any blocks. Google refreshes the data every few hours automatically — re-running Sync pulls the latest.

Step 1 · Get your iCal feed URL from Schoology

  1. Log in to Schoology on a computer (easier than mobile).
  2. Click the Calendar icon in the top bar (looks like 📅).
  3. Look at the bottom-right corner of the calendar page for the iCal Feed link. On some schools it's under a ≡ menu or a small gear icon instead.
  4. Click it — a box pops up showing your personal feed URL. It looks something like:
    https://yourschool.schoology.com/iCal?user=...&token=...
  5. Copy the entire URL.
Can't find iCal Feed? Some schools disable the link. If so, skip to Method 2 below.

Step 2 · Method 1 — Paste the URL (easiest)

  1. In StudyDeck, click 📥 Import from Schoology (top of Dashboard or Assignments).
  2. Paste your URL into the "Option 1 — iCal feed URL" box.
  3. Click Fetch feed. StudyDeck will download the calendar.
  4. Click Parse & import.
  5. Done! Your assignments appear in the Assignments tab, tagged 🎓 Schoology.
Tip: Save the URL in Settings → Schoology import, then use 🔄 Sync now anytime you want to pull fresh assignments.

Step 2 · Method 2 — Paste the file contents (if fetching fails)

Some schools block outside apps from reading the feed directly (a CORS restriction). No problem — you can download it yourself and paste the contents.

  1. Paste your iCal URL directly into your browser's address bar and press Enter.
  2. One of two things will happen:
    • Your browser shows text starting with BEGIN:VCALENDAR — select all (Ctrl/Cmd+A), copy (Ctrl/Cmd+C).
    • Your browser downloads a .ics file — open it with a text editor (Notepad, TextEdit) and copy everything.
  3. In StudyDeck, open the import dialog and paste the copied text into "Option 2 — Paste .ics contents".
  4. Click Parse & import.

Step 3 · What gets imported

  • Title — the assignment name from Schoology
  • Due date & time — whatever Schoology has set
  • Class — auto-detected from the event's category. If it's new, a class is created with a random color. You can rename it and change the color in Settings → Classes.
  • Description/notes — copied into the Notes field
  • Priority — defaults to Medium. Adjust per-assignment.

Imported items get a 🎓 Schoology tag. You can still add subtasks, notes, and priorities on top — those stay put on re-sync.

Keeping things in sync

Schoology doesn't push updates — you have to pull. A few approaches:

  • Manual: Click 🔄 Sync now in Settings whenever you want fresh data. Re-imports skip duplicates.
  • Routine: Get in the habit of syncing once in the morning and once on Sunday evening.
  • Deleted in Schoology? StudyDeck won't delete tasks you already have — delete them manually if needed.

Troubleshooting

❓ "Fetch failed"
Your school blocks cross-origin requests. Use Method 2 (paste the text) instead.

❓ "No events found"
You probably pasted something that isn't an iCal file. Make sure the text starts with BEGIN:VCALENDAR.

❓ I don't see an iCal Feed link in Schoology
Your school admin may have disabled it. Contact a teacher or try logging in from a different account view (student vs. parent).

❓ Assignments are missing due dates
Events in Schoology without a due date (like announcements) are still imported but show as "No due date" — you can add a date manually.

❓ Imported to the wrong class
Open the assignment, click edit (✎), and change the class. You can also rename classes in Settings.

Run StudyDeck from a local server 🖥️

Pick any one method — they all do the same thing. You only do this once.

Why this is needed: When you open study-planner.html by double-clicking it, your browser treats it as an untrusted local file (file:///...) and blocks it from talking to external services like Google Apps Script. Running a tiny local server changes the URL to http://localhost:... which the browser trusts. Nothing is uploaded anywhere — the "server" just serves files from your own computer.

⭐ Method 1 — Python (simplest if you have it)

Python comes pre-installed on Mac & most Linux. On Windows, install it from python.org first (tick "Add to PATH" during install).

  1. Open Terminal (Mac/Linux) or Command Prompt (Windows)
  2. Navigate to the Study Planner folder: 
    cd "path/to/Study Planner"
  3. Run:
    python -m http.server 8000
    (Try python3 if python doesn't work)
  4. Open in your browser: http://localhost:8000/study-planner.html

Stop the server by pressing Ctrl+C in the terminal.

🟢 Method 2 — Node.js (if you have it)

  1. Open Terminal / Command Prompt in the Study Planner folder
  2. Run:
    npx serve
    (First time it'll ask to install — say yes)
  3. Visit the http://localhost:3000 URL it prints

🔵 Method 3 — VS Code (if you use it)

  1. Install the Live Server extension (by Ritwick Dey)
  2. Open the Study Planner folder in VS Code
  3. Right-click study-planner.html → Open with Live Server
  4. Your browser opens automatically at http://127.0.0.1:5500

🌐 Method 4 — GitHub Pages (permanent, access from anywhere)

  1. Create a free GitHub account if you don't have one
  2. Make a new public repo
  3. Upload study-planner.html and rename it to index.html
  4. Go to repo Settings → Pages → set Source to main branch
  5. Wait ~1 minute, then visit https://yourusername.github.io/reponame/

Bonus: your planner is now accessible from any device at that URL. Data still stays per-browser though.

How do I know it worked?

Look at your browser's address bar. If it starts with http:// or https:// — you're good, sync will work.
If it still says file:/// — the browser is still loading the file directly. Double-check you opened it via the http://localhost:... URL, not by double-clicking the file.
